# 沃林学生管理系统 - 表设计文档

**版本**: v1.0.0  
**更新日期**: 2025年  
**状态**: 正式版

---

## 一、数据库概述

### 1.1 基本信息

| 属性 | 值 |
|------|-----|
| 数据库类型 | MySQL 8.0+ |
| ORM框架 | SQLAlchemy 2.0 |
| 字符集 | utf8mb4 |
| 排序规则 | utf8mb4_unicode_ci |
| 连接池大小 | 5 |
| 连接池回收周期 | 3600秒 |

### 1.2 设计原则

- **逻辑删除**：所有数据表包含 `is_deleted` 字段，支持软删除
- **外键关联**：表间通过外键建立关联关系
- **时间戳**：涉及时间的数据表包含创建/更新时间字段
- **统一命名**：使用下划线命名法（snake_case）

## 三、表结构详细设计

### 3.1 用户表 (users)

系统用户表，用于身份认证和权限管理。

| 字段名 | 数据类型 | 约束 | 默认值 | 说明 |
|--------|----------|------|--------|------|
| id | INT | PRIMARY KEY, AUTO_INCREMENT | - | 用户ID |
| username | VARCHAR(50) | UNIQUE, NOT NULL, INDEX | - | 用户名 |
| hashed_password | VARCHAR(255) | NOT NULL | - | 加密密码 |
| is_active | BOOLEAN | NOT NULL | TRUE | 是否激活 |
| role | VARCHAR(20) | NOT NULL | 'student' | 角色：admin/teacher/student |
| student_id | INT | FOREIGN KEY | NULL | 关联学生ID |
| teacher_id | INT | FOREIGN KEY | NULL | 关联教师ID |

**索引**：
- PRIMARY KEY (`id`)
- UNIQUE INDEX (`username`)

**关系**：
- `student_id` → `stu_basic_info.stu_id`（学生用户绑定）
- `teacher_id` → `teacher.teacher_id`（教师用户绑定）

**业务规则**：
- 用户注册默认为学生角色，账户禁用状态
- 管理员和教师账号禁止公开注册

---

### 3.2 教师表 (teacher)

存储教师/讲师信息。

| 字段名 | 数据类型 | 约束 | 默认值 | 说明 |
|--------|----------|------|--------|------|
| teacher_id | INT | PRIMARY KEY, AUTO_INCREMENT | - | 教师ID |
| teacher_name | VARCHAR(30) | COMMENT | - | 姓名 |
| gender | VARCHAR(10) | COMMENT | - | 性别：男/女 |
| phone | VARCHAR(20) | COMMENT | - | 联系电话 |
| role | VARCHAR(20) | COMMENT | - | 角色：counselor(顾问)/headteacher(班主任)/lecturer(讲师) |
| is_deleted | BOOLEAN | NOT NULL | FALSE | 逻辑删除：0-未删除，1-已删除 |

**索引**：
- PRIMARY KEY (`teacher_id`)

**关系**：
- 一对多：教师可担任多个班级的班主任 (`class.head_teacher_id`)
- 多对多：通过 `class_teacher` 中间表关联多个班级
- 一对多：教师可担任多个学生的顾问 (`stu_basic_info.advisor_id`)

---

### 3.3 班级表 (class)

存储班级信息。

| 字段名 | 数据类型 | 约束 | 默认值 | 说明 |
|--------|----------|------|--------|------|
| class_id | INT | PRIMARY KEY, AUTO_INCREMENT | - | 班级编号 |
| class_name | VARCHAR(50) | NOT NULL | - | 班级名称 |
| start_time | DATETIME | NOT NULL | - | 开课时间 |
| is_deleted | BOOLEAN | NOT NULL | FALSE | 逻辑删除 |
| head_teacher_id | INT | FOREIGN KEY | NULL | 班主任ID |

**索引**：
- PRIMARY KEY (`class_id`)
- FOREIGN KEY (`head_teacher_id`) → `teacher.teacher_id`

**关系**：
- 一对多：一个班级有多个学生 (`stu_basic_info`)
- 多对多：通过 `class_teacher` 关联多个教师
- 一对多：一个班级有多条就业记录 (`employment`)

---

### 3.4 班级-教师关联表 (class_teacher)

班级与教师的多对多关联表。

| 字段名 | 数据类型 | 约束 | 说明 |
|--------|----------|------|------|
| class_id | INT | PRIMARY KEY, FOREIGN KEY | 班级ID |
| teacher_id | INT | PRIMARY KEY, FOREIGN KEY | 教师ID |

**索引**：
- PRIMARY KEY (`class_id`, `teacher_id`)
- FOREIGN KEY (`class_id`) → `class.class_id`
- FOREIGN KEY (`teacher_id`) → `teacher.teacher_id`

---

### 3.5 学生基本信息表 (stu_basic_info)

存储学生个人信息。

| 字段名 | 数据类型 | 约束 | 默认值 | 说明 |
|--------|----------|------|--------|------|
| stu_id | INT | PRIMARY KEY, AUTO_INCREMENT | - | 学生ID |
| stu_name | VARCHAR(20) | NOT NULL | - | 学生姓名 |
| native_place | VARCHAR(50) | NOT NULL | - | 籍贯 |
| graduated_school | VARCHAR(50) | NOT NULL | - | 毕业学校 |
| major | VARCHAR(50) | NOT NULL | - | 专业 |
| admission_date | DATETIME | NOT NULL | - | 入学日期 |
| graduation_date | DATETIME | NOT NULL | - | 毕业日期 |
| education | VARCHAR(20) | NOT NULL | - | 学历 |
| age | INT | NOT NULL | - | 年龄 |
| gender | VARCHAR(2) | NOT NULL | - | 性别 |
| is_deleted | BOOLEAN | NOT NULL | FALSE | 逻辑删除 |
| advisor_id | INT | FOREIGN KEY | - | 导师/顾问ID |
| class_id | INT | FOREIGN KEY, NOT NULL | - | 班级ID |

**索引**：
- PRIMARY KEY (`stu_id`)
- FOREIGN KEY (`advisor_id`) → `teacher.teacher_id`
- FOREIGN KEY (`class_id`) → `class.class_id`

**关系**：
- 多对一：学生属于一个班级 (`class`)
- 多对一：学生有一个导师 (`teacher`)
- 一对多：学生有多条成绩记录 (`stu_exam_record`)
- 一对多：学生有多条就业记录 (`employment`)

**业务规则**：
- `class_id` 字段必须关联有效班级
- 查询时必须过滤 `is_deleted = 0`

---

### 3.6 学生成绩记录表 (stu_exam_record)

存储学生各次考核成绩。

| 字段名 | 数据类型 | 约束 | 默认值 | 说明 |
|--------|----------|------|--------|------|
| stu_id | INT | PRIMARY KEY, FOREIGN KEY | - | 学生ID |
| seq_no | INT | PRIMARY KEY | - | 考核序次/学期 |
| grade | INT | DEFAULT 0 | 0 | 考核成绩(0-100) |
| exam_date | DATE | DEFAULT '1900-01-01' | '1900-01-01' | 考核日期 |
| is_deleted | INT | DEFAULT 0 | 0 | 逻辑删除：0-未删除，1-已删除 |

**索引**：
- PRIMARY KEY (`stu_id`, `seq_no`) - 复合主键
- FOREIGN KEY (`stu_id`) → `stu_basic_info.stu_id`

**业务规则**：
- 同一学生同一考核序次只有一条记录
- 成绩范围：0-100分
- 查询时必须过滤 `is_deleted = 0`

---

### 3.7 学生就业信息表 (employment)

存储学生就业信息。

| 字段名 | 数据类型 | 约束 | 默认值 | 说明 |
|--------|----------|------|--------|------|
| emp_id | INT | PRIMARY KEY, AUTO_INCREMENT | - | 就业记录ID |
| stu_id | INT | FOREIGN KEY, NOT NULL | - | 学生ID |
| stu_name | VARCHAR(20) | NOT NULL | - | 学生姓名（冗余字段） |
| class_id | INT | FOREIGN KEY, NOT NULL | - | 班级ID |
| open_time | DATE | - | - | 就业开放时间 |
| offer_time | DATE | - | - | 收到offer时间 |
| company | VARCHAR(50) | - | - | 就业公司 |
| salary | FLOAT | - | - | 薪资（单位：K） |
| is_deleted | BOOLEAN | DEFAULT FALSE | FALSE | 逻辑删除 |

**索引**：
- PRIMARY KEY (`emp_id`)
- FOREIGN KEY (`stu_id`) → `stu_basic_info.stu_id`
- FOREIGN KEY (`class_id`) → `class.class_id`

**业务规则**：
- 根据 `student_id` 判断是否存在就业记录，存在则更新，不存在则创建
- 薪资单位为千元（K）
- 查询时必须过滤 `is_deleted = FALSE`

**索引**：
- PRIMARY KEY (`id`)

## 六、命名规范

### 6.1 表名命名

| 规范 | 示例 |
|------|------|
| 使用下划线命名 | `stu_basic_info`, `class_teacher` |
| 单数形式 | `class` 而非 `classes` |
| 有意义的名称 | `employment` 而非 `emp` |

### 6.2 字段命名

| 类型 | 规范 | 示例 |
|------|------|------|
| 主键 | `表名_id` 或 `id` | `stu_id`, `teacher_id` |
| 外键 | `表名_id` | `class_id`, `stu_id` |
| 布尔值 | `is_` 前缀 | `is_deleted`, `is_active` |
| 时间 | `_time` 或 `_date` 后缀 | `start_time`, `exam_date` |

## 七、初始化SQL

### 7.1 核心表创建

```sql
-- 用户表
CREATE TABLE users (
    id INT PRIMARY KEY AUTO_INCREMENT,
    username VARCHAR(50) UNIQUE NOT NULL,
    hashed_password VARCHAR(255) NOT NULL,
    is_active BOOLEAN DEFAULT TRUE,
    role VARCHAR(20) DEFAULT 'student',
    student_id INT,
    teacher_id INT,
    INDEX idx_username (username)
);

-- 教师表
CREATE TABLE teacher (
    teacher_id INT PRIMARY KEY AUTO_INCREMENT,
    teacher_name VARCHAR(30),
    gender VARCHAR(10),
    phone VARCHAR(20),
    role VARCHAR(20),
    is_deleted BOOLEAN DEFAULT FALSE
);

-- 班级表
CREATE TABLE class (
    class_id INT PRIMARY KEY AUTO_INCREMENT,
    class_name VARCHAR(50) NOT NULL,
    start_time DATETIME NOT NULL,
    is_deleted BOOLEAN DEFAULT FALSE,
    head_teacher_id INT,
    FOREIGN KEY (head_teacher_id) REFERENCES teacher(teacher_id)
);

-- 班级-教师关联表
CREATE TABLE class_teacher (
    class_id INT,
    teacher_id INT,
    PRIMARY KEY (class_id, teacher_id),
    FOREIGN KEY (class_id) REFERENCES class(class_id),
    FOREIGN KEY (teacher_id) REFERENCES teacher(teacher_id)
);

-- 学生基本信息表
CREATE TABLE stu_basic_info (
    stu_id INT PRIMARY KEY AUTO_INCREMENT,
    stu_name VARCHAR(20) NOT NULL,
    native_place VARCHAR(50) NOT NULL,
    graduated_school VARCHAR(50) NOT NULL,
    major VARCHAR(50) NOT NULL,
    admission_date DATETIME NOT NULL,
    graduation_date DATETIME NOT NULL,
    education VARCHAR(20) NOT NULL,
    age INT NOT NULL,
    gender VARCHAR(2) NOT NULL,
    is_deleted BOOLEAN DEFAULT FALSE,
    advisor_id INT,
    class_id INT NOT NULL,
    FOREIGN KEY (advisor_id) REFERENCES teacher(teacher_id),
    FOREIGN KEY (class_id) REFERENCES class(class_id)
);

-- 学生成绩记录表
CREATE TABLE stu_exam_record (
    stu_id INT,
    seq_no INT,
    grade INT DEFAULT 0,
    exam_date DATE DEFAULT '1900-01-01',
    is_deleted INT DEFAULT 0,
    PRIMARY KEY (stu_id, seq_no),
    FOREIGN KEY (stu_id) REFERENCES stu_basic_info(stu_id)
);

-- 就业信息表
CREATE TABLE employment (
    emp_id INT PRIMARY KEY AUTO_INCREMENT,
    stu_id INT NOT NULL,
    stu_name VARCHAR(20) NOT NULL,
    class_id INT NOT NULL,
    open_time DATE,
    offer_time DATE,
    company VARCHAR(50),
    salary FLOAT,
    is_deleted BOOLEAN DEFAULT FALSE,
    FOREIGN KEY (stu_id) REFERENCES stu_basic_info(stu_id),
    FOREIGN KEY (class_id) REFERENCES class(class_id)
);
```
