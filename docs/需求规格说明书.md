## 一、引言

### 1.1 项目背景

「拾光学子成长管理平台」是一套面向教育培训机构的全方位学生管理解决方案。随着教育机构规模扩大，传统手工记录与 Excel 管理方式已经难以满足学员从入学到就业的全生命周期管理需求。本项目旨在构建一个集学生信息管理、考核成绩追踪、就业管理、统计分析及 AI 智能服务于一体的综合性平台，实现教育管理的数字化、智能化转型。

### 1.2 项目目标

| 目标 | 描述 |
|------|------|
| 管理数字化 | 实现学生、班级、教师信息的规范化录入与维护，替代纸质/Excel 管理 |
| 过程可视化 | 提供多维度统计分析，生成数据仪表盘，辅助教学决策 |
| 服务智能化 | 引入大模型能力，支持自然语言查库、知识库问答、智能对话 |
| 体验现代化 | 提供 Web 可视化前端界面，降低使用门槛 |

### 1.3 适用范围

本系统适用于中小型教育培训机构、职业培训学校等需要管理学员信息、追踪学习进度、统计就业情况的场景。

### 1.4 术语与缩写

| 术语 | 全称 / 解释 |
|------|------------|
| CRUD | Create, Read, Update, Delete（增删改查） |
| JWT | JSON Web Token，用于用户认证与授权 |
| RAG | Retrieval-Augmented Generation，检索增强生成 |
| NL2SQL | Natural Language to SQL，自然语言转 SQL 查询 |
| SSE | Server-Sent Events，服务端推送（用于 AI 流式输出） |
| ORM | Object Relational Mapping，对象关系映射 |
| 逻辑删除 | 不物理删除数据行，仅将 `is_deleted` 字段标记为已删除 |

### 3.1 用户认证模块 (AUTH)

> **路由**: `/auth`  
> **实现**: `controllers/auth_controller.py` + `services/auth_service.py`

#### 3.1.1 用户注册

| 需求ID | 需求标题 | 优先级 |
|--------|----------|--------|
| AUTH-R01 | 支持用户名 + 密码注册，角色默认为 student | P0 |
| AUTH-R02 | 注册后账户状态为禁用（is_active=false），需管理员审核后才能登录 | P0 |
| AUTH-R03 | 禁止通过注册接口创建 admin / teacher 角色 | P0 |
| AUTH-R04 | 用户名校验：3~50 字符，仅允许字母、数字、下划线 | P0 |
| AUTH-R05 | 密码强度校验：最小长度 8 位，必须包含大小写字母和数字 | P0 |
| AUTH-R06 | 用户名全局唯一，重复注册返回 409 错误 | P0 |

#### 3.1.2 用户登录

| 需求ID | 需求标题 | 优先级 |
|--------|----------|--------|
| AUTH-L01 | 支持用户名 + 密码登录 | P0 |
| AUTH-L02 | 登录成功返回 JWT access_token（有效期 7 天） | P0 |
| AUTH-L03 | 密码使用 bcrypt 哈希存储，严禁明文 | P0 |
| AUTH-L04 | 已禁用（is_active=false）的账户禁止登录 | P0 |
| AUTH-L05 | 提供获取当前登录用户信息的接口 | P0 |

**接口清单**：

| 方法 | 端点 | 说明 | 认证 |
|------|------|------|------|
| POST | `/auth/register` | 用户注册 | 否 |
| POST | `/auth/login` | 用户登录 | 否 |
| GET | `/auth/me` | 获取当前用户信息 | 是 |

| STU-C03 | 创建学生时可指定指导教师/顾问（advisor_id 选填，角色须为 counselor） | P1 |
| STU-C04 | 创建学生时自动级联创建空白就业记录 | P0 |
| STU-R01 | 查询学生列表，支持按学号、姓名、班级筛选 | P0 |
| STU-R02 | 查询单个学生详情 | P0 |
| STU-R03 | 查询结果自动过滤已逻辑删除的记录 | P0 |
| STU-U01 | 更新学生信息，支持部分字段更新 | P0 |
| STU-U02 | 更新 advisor_id 时验证目标教师角色为 counselor | P1 |
| STU-D01 | 逻辑删除学生（设置 is_deleted = true），不物理删除 | P0 |
| STU-D02 | 仅管理员可执行删除操作 | P0 |

#### 学生字段详情

| 字段名 | 数据库类型 | 必填 | 说明 | 校验规则 |
|--------|-----------|------|------|----------|
| stu_name | VARCHAR(20) | 是 | 姓名 | 1-20 字符 |
| gender | VARCHAR(2) | 是 | 性别 | 值必须为"男"或"女" |
| age | INT | 是 | 年龄 | 1-150 |
| native_place | VARCHAR(50) | 是 | 籍贯 | 最大 50 字符 |
| graduated_school | VARCHAR(50) | 是 | 毕业院校 | 最大 50 字符 |
| major | VARCHAR(50) | 是 | 专业 | 最大 50 字符 |
| education | VARCHAR(20) | 是 | 学历 | 最大 20 字符 |
| admission_date | DATETIME | 是 | 入学日期 | 合法日期 |
| graduation_date | DATETIME | 是 | 毕业日期 | 须晚于入学日期 |
| class_id | INT | 是 | 班级ID | 须存在于 class 表 |
| advisor_id | INT | 否 | 导师ID | 须存在于 teacher 表且 role=counselor |

**接口清单**：

| 方法 | 端点 | 说明 | 认证 | 授权 |

| GET | `/students/` | 查询学生列表（支持 ?stu_id=&stu_name=&class_id=） | 是 | admin/teacher |

### 3.10 NL2SQL 自然语言查询模块 (NL2)

> **路由**: `/query`  
> **实现**: `controllers/query_controller.py` + `config/db_schema.py`  
> **依赖**: Kimi 大模型

#### 3.10.1 核心流程

```
用户自然语言问题 → 检索表结构上下文 → Kimi 生成 SQL → SQL 安全校验 → 执行查询 → 返回结果
```

| 需求ID | 需求标题 | 优先级 |
|--------|----------|--------|
| NL2-001 | 支持用户使用自然语言查询数据库 | P0 |
