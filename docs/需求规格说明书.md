## 一、项目概述

### 1.1 项目背景

沃林学生管理系统是一个面向教育机构的综合性管理平台，集成了学生信息管理、成绩管理、就业跟踪、统计分析以及人工智能辅助功能。系统旨在提高教育管理效率，为学生提供智能化服务。

### 1.2 项目目标

1. **管理效率提升**：实现学生信息的数字化、规范化管理
2. **数据驱动决策**：提供多维度的统计分析功能
3. **智能化服务**：引入AI技术，提供智能问答、自然语言查询等能力
4. **用户体验优化**：提供友好的前端界面和流畅的操作体验

### 1.3 术语定义

| 术语 | 定义 |
|------|------|
| CRUD | Create, Read, Update, Delete - 增删改查操作 |
| JWT | JSON Web Token - 用于身份认证的令牌 |
| RAG | Retrieval-Augmented Generation - 检索增强生成 |
| NL2SQL | Natural Language to SQL - 自然语言转SQL |
| 逻辑删除 | 数据不物理删除，仅标记is_deleted字段 |

## 五、接口约束

### 5.1 响应格式约束

所有API必须遵循统一响应格式：

```json
{
    "code": 200,       // 业务状态码
    "message": "success",
    "data": {}
}
```

### 5.2 错误处理约束

| 场景 | 处理方式 |
|------|---------|
| 参数校验失败 | HTTP 400 + detail字段 |
| 未认证 | HTTP 401 + detail字段 |
| 无权限 | HTTP 403 + detail字段 |
| 资源不存在 | HTTP 404 + detail字段 |
| 服务器错误 | HTTP 500 + 通用错误信息 |
